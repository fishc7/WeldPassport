# WeldPassport — Процессы v0.1

Сводная карта процессов сварочного производства (межролевой обзор). Детализация по
ролям — в `05_Роли_и_права/<роль>/Процессы/`. Жизненный цикл системы — в
`../docs/00_PROJECT_CONTEXT.md`.

## 1. Подготовка сварщика

**Цель:** определить, может ли конкретный работник быть допущен к сварке на объекте.

### Шаги

1. Принять работника на работу.
2. Внести карточку работника.
3. Зафиксировать документы: НАКС, свидетельство, диплом, обучение.
4. Провести внутреннюю проверку / внутреннюю аттестацию.
5. Оформить внутренний допуск.
6. При необходимости оформить допуск заказчика.
7. Разрешить сварку на производстве только в пределах допуска.

## 2. Подготовка ПТО

**Цель:** подготовить рабочую документацию для выполнения сварки.

### Шаги

1. Создать проект и объект.
2. Внести изометрии.
3. Пронумеровать изометрии и стыки.
4. Внести номенклатуру деталей трубопровода.
5. Связать стыки с проектом, объектом и изометрией.
6. Передать стыки в работу.

## 3. Фиксация фактической сварки

**Цель:** зафиксировать, кто реально варил конкретный стык.

### Шаги

1. Мастер/прораб получает перечень стыков в работе.
2. Указывает фактического сварщика.
3. При необходимости указывает звеньевого/помощника.
4. Указывает дату, смену, способ сварки, материал.
5. Система проверяет допуск сварщика.
6. Запись передается ПТО для исполнительной документации.

## 4. Контроль и НК

**Цель:** связать результат контроля с конкретным стыком и исполнителем.

### Шаги

1. Назначить вид контроля.
2. Зафиксировать результат НК.
3. Зафиксировать дефекты при наличии.
4. Назначить ремонт.
5. Повторно проверить после ремонта.
6. Передать данные в исполнительный журнал.

## 5. Учет сварочных материалов

**Цель:** связать выдачу и расход материалов с работами.

### Шаги

1. Внести партию материала и сертификат.
2. Зафиксировать выдачу сварщику/звену/участку.
3. Связать расход с проектом, объектом, стыком или сменой.
4. Сравнить норму и фактический расход.
