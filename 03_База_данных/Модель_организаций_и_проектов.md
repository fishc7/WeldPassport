# Модель организаций и проектов

> Сводный узел графа. Каноническое описание — [[docs/ARCHITECTURE#Модель организаций и проектов|Архитектура]];
> обоснование решения — [[docs/project/DECISIONS#ADR-001. Модель организаций и проектов|ADR-001]].
> Здесь — краткая карта связей без дублирования канона.

## Что это

Связь организаций и проектов в WeldPassport — **многие ко многим**. Правило
«1 фирма = 1 проект» **не используется**: одна организация может участвовать во
многих проектах, один проект может включать несколько организаций. Роль организации
в конкретном проекте фиксируется в связующей таблице `project_companies`.

```text
companies ↔ projects   (через project_companies)
```

## Сущности

| Сущность | Назначение |
|---|---|
| `companies` | Справочник организаций/фирм |
| `projects` | Справочник проектов/объектов |
| `project_companies` | Связь организация↔проект; поле `role` — роль организации в проекте |

Роли организации (`role`): `customer`, `general_contractor`, `welding_contractor`,
`ndt_lab`, `inspection`, `designer`.

## Иерархия данных

```text
Организация / фирма ↕ Проект / объект → Участок / линия → Изометрия / чертеж → Стык
```

Ниже уровня проекта (участок → изометрия → стык) — производственный контур; он не
смешивается с ролями организаций в проекте.
