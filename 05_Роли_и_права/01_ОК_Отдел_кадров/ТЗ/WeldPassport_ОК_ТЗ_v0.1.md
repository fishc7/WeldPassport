# WeldPassport — Модуль ОК (Отдел кадров). ТЗ v0.1

## 1. Роль ОК в системе

ОК ведёт реестр работников предприятия. Зона ответственности: от приёма на работу до передачи работника в зону Главного сварщика (допуски к сварке).

**ОК делает:**
- Заводит карточку работника при приёме.
- Присваивает должность из справочника ЕТКС.
- Фиксирует статус занятости (работает / уволен / в отпуске).
- Хранит кадровые документы (по типу).

**ОК НЕ делает:**
- Не выдаёт сварочные допуски — это Главный сварщик.
- Не аттестует сварщиков — это Главный сварщик.
- Не назначает сварщиков на стыки — это Мастер/ПТО.

## 2. Процессы ОК

### 2.1 Приём работника

1. Создать карточку работника (ФИО, табельный номер, должность, организация).
2. Установить дату приёма и статус «Работает».
3. Добавить кадровые документы при наличии.
4. Сохранить запись — работник доступен остальным ролям.

### 2.2 Увольнение работника

1. Найти карточку работника.
2. Установить статус «Уволен» и дату увольнения.
3. Работник остаётся в реестре (история), но недоступен для новых назначений.

### 2.3 Поиск и просмотр работника

1. Поиск по ФИО или табельному номеру.
2. Фильтр по статусу (работает / уволен) и должности.
3. Просмотр карточки: данные + список документов.

### 2.4 Редактирование карточки

1. Изменить ФИО, должность, организацию, табельный номер.
2. Изменить статус занятости.

## 3. Поля карточки работника

Маппинг на текущую модель `Rabotnik` (таблица `РАБОТНИКИ`):

| Поле в системе       | Колонка в БД        | Тип        | Обязательное | Примечание                              |
|----------------------|---------------------|------------|--------------|-----------------------------------------|
| ФИО                  | ФИО                 | String(255)| Да           |                                         |
| Табельный номер      | Табельный_Номер     | String(50) | Нет          | Уникальный в рамках организации?        |
| Должность            | ID_Должности        | FK → ЕТКС  | Нет          | Выбор из справочника ЕТКС               |
| Организация          | Организация         | String(255)| Нет          | Субподрядчик или собственное предприятие|
| Дата приёма          | Дата_Приема         | Date       | Нет          |                                         |
| Статус               | Статус              | String(50) | Нет          | «Работает» / «Уволен» / …              |

> Строковое поле `Должность` (старый формат) сохранено для переходного периода. После миграции данных — убрать.

## 4. Открытые вопросы

- [ ] **Кадровые документы работника** — нужен ли отдельный раздел «Документы ОК» (трудовой договор, паспорт, диплом)? Или документы ведёт только Главный сварщик (НАКС, свидетельство)?
- [ ] **Организация** — это справочник или свободная строка? Субподрядчики меняются — нужен ли реестр организаций?
- [ ] **Статус занятости** — какой набор статусов? (Работает / Уволен / В отпуске / Командирован / …)
- [ ] **Уникальность табельного номера** — в рамках одной организации или глобально по системе?
- [ ] **Права доступа** — ОК только читает чужие данные (Главного сварщика) или полностью изолирован?
- [ ] **Дата увольнения** — нужно отдельное поле, или достаточно статуса «Уволен»?
- [ ] **Поиск** — нужен ли глобальный поиск по всем ролям, или каждый ищет в своём разделе?
