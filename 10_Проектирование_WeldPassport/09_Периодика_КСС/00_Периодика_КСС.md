# Периодика КСС

> Тематический узел (черновик проектирования). Правила партий — по ГОСТ 32569;
> план модуля — [[docs/project/ROADMAP#7. Спроектировать модуль периодики КСС|ROADMAP §7]];
> рабочий чат — [[docs/project/CHAT_INDEX#6. Периодика КСС|CHAT_INDEX §6]].

## Что это

КСС — контрольные сварные соединения (периодическая проверка стабильности сварки).
Модуль периодики КСС отслеживает, для каких партий однотипных стыков требуется
периодический контроль, и формирует эти партии по правилам нормативов.

## Зачем это нужно

Периодический контроль подтверждает, что процесс сварки остаётся стабильным на
протяжении производства, а не только на отдельном стыке. Это основа для корректного
объёма НК и для исполнительной документации.

## Связь с ГОСТ 32569

Партия однотипных стыков формируется по правилам ГОСТ 32569 (см. [[docs/project/ROADMAP#7. Спроектировать модуль периодики КСС|ROADMAP §7]]):

- партия — однотипные стыки;
- срок партии — **не более 3 месяцев**;
- лимит количества стыков по группам **DN** (диаметр);
- один сварщик на партию;
- одна марка стали;
- единый процесс сварки;
- толщина в пределах **±50 %**;
- **удвоение объёма контроля** при обнаружении брака.

## Связанные сущности БД

- `joints` — стыки, входящие в партию;
- `weld_operations` — сварочные операции по стыкам партии;
- `kss_parties` — сами партии КСС (границы, срок, лимиты, сварщик, марка, процесс);
- `ndt_reports` — результаты контроля партии и правило удвоения.

Канон модели данных — [[docs/ARCHITECTURE|Архитектура]] и [[03_База_данных/WeldPassport_Реестр_сущностей_БД_v0.1|Реестр сущностей БД]].
