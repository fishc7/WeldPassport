# WeldPassport — Правила проекта

**Версия:** 1.0  
**Дата:** 2026-07-01

---

# 1. Назначение проекта

WeldPassport — это промышленная информационная система управления сварочным производством.

Проект объединяет функции:

- MES
- ERP
- Электронного журнала сварки
- Управления персоналом
- Исполнительной документации
- Контроля качества
- Аттестации сварщиков
- Нормирования
- Производственной аналитики

Главная сущность системы — **Сварной стык**.

---

# 2. Главные принципы

Проект разрабатывается как долгосрочная промышленная система.

Основные требования:

✔ Простая архитектура

✔ Масштабируемость

✔ Минимум дублирования

✔ Максимальная читаемость

✔ Документирование всех решений

✔ Все изменения проходят через Git

---

# 3. Источник истины

Для каждой информации существует только одно место хранения.

| Информация | Источник |
|------------|----------|
| Архитектура | docs |
| Код Backend | 09_Разработка/backend |
| Код Frontend | 09_Разработка/frontend |
| Desktop приложения | 09_Разработка/desktop_ok |
| Справочники | 08_Справочники |
| Паспорт проекта | 00_Паспорт_проекта |

Запрещается создавать дубли документов.

---

# 5. Backend

Основной Backend располагается

```
09_Разработка/backend
```

Используется

- FastAPI

- SQLAlchemy

- Alembic

- PostgreSQL

Backend является единственным местом бизнес-логики.

Запрещается размещать бизнес-логику во Frontend.

---

# 6. Frontend

Frontend располагается

```
09_Разработка/frontend
```

Используются

- React

- TypeScript

- Vite

Frontend отвечает только за отображение данных.

---

# 7. Desktop

Desktop приложения допускаются только как отдельные специализированные рабочие места.

Например

```
desktop_ok
```

Каждое Desktop приложение имеет собственный README.

---

# 8. База данных

База данных является единым источником данных.

Все изменения выполняются только через миграции Alembic.

Запрещается изменять структуру базы вручную.

---

```

```

---

# 11. Документация

Каждая новая подсистема должна иметь описание.

Если появляется новая бизнес-логика —

обязательно обновляется документация.

---

# 12. Справочники

Все нормативные данные располагаются только в

```
08_Справочники
```

Исходные Excel не изменяются.

Импорт выполняется через отдельные скрипты.

---

# 13. AI-ассистенты

Допускается использование:

- ChatGPT

- Cursor

- Claude

ИИ не принимает архитектурные решения самостоятельно.

Все архитектурные изменения должны быть подтверждены владельцем проекта.

---

# 14. Code Review

Перед слиянием проверяется

- архитектура

- стиль

- читаемость

- отсутствие дублирования

- документация

---

# 15. Definition of Done

Задача считается завершённой если

✔ код работает

✔ написаны тесты (если применимо)

✔ обновлена документация

✔ выполнен коммит

✔ отсутствуют ошибки линтера

✔ отсутствуют дубли

---

# 16. Запрещается

Запрещается

- хранить секреты в Git

- коммитить node_modules

- коммитить dist

- изменять БД вручную

- дублировать документы

- создавать папки без назначения
