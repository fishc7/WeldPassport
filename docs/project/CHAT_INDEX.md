# WeldPassport — индекс рабочих чатов

## Назначение

Этот файл фиксирует рабочие чаты и обсуждения по проекту WeldPassport.

Он нужен, чтобы не терять важные мысли, решения и договорённости при росте количества чатов.

Чаты используются как рабочие обсуждения.

Источником истины являются проектные файлы:

- [[docs/ARCHITECTURE|docs/ARCHITECTURE]]
- [[docs/project/DECISIONS|docs/project/DECISIONS]]
- [[docs/project/PROJECT_SUMMARY|docs/project/PROJECT_SUMMARY]]
- [[docs/project/ROADMAP|docs/project/ROADMAP]]
- [[AGENTS|AGENTS]]

## Главное правило

Если решение есть только в чате — оно ещё не считается закреплённым.

Решение считается закреплённым только после переноса в проектную документацию.

Правило работы:

```text
Обсудили в чате → приняли решение → перенесли в документацию → проверили git diff → сделали commit
```

---

## Статусы разбора чатов

Используются статусы:

```text
new          — чат ещё не разобран
in_progress  — из чата извлекаются решения
documented   — решения перенесены в документацию
archived     — чат сохранён как история, активных действий нет
```

## Рабочие чаты

### 1. Архитектура WeldPassport

Статус: `in_progress`

Назначение:

Обсуждение общей архитектуры WeldPassport, базовых сущностей, ролей, проектных решений и правил работы AI-агентов.

Ключевые темы:

- модель организаций и проектов;
- проектные решения и ADR;
- структура документации;
- правила фиксации решений;
- контроль Cursor Agent и Claude Code;
- разделение ответственности между ChatGPT, Cursor Agent и Claude Code.

Ключевые решения:

- не использовать модель «1 фирма = 1 проект»;
- использовать связь `companies ↔ projects` через `project_companies`;
- решения фиксировать в `docs/project/DECISIONS.md`;
- текущую архитектуру фиксировать в `docs/ARCHITECTURE.md`;
- правила для AI-агентов фиксировать в `AGENTS.md`;
- чат не считать единственным источником истины.

Связанные файлы:

- `docs/ARCHITECTURE.md`
- `docs/project/DECISIONS.md`
- `docs/project/PROJECT_SUMMARY.md`
- `docs/project/ROADMAP.md`
- `AGENTS.md`

Что ещё проверить:

- перенесена ли ролевая цепочка `ОК → ОГС → СМР → ПТО → ОТК → Закрытие` в `PROJECT_SUMMARY.md`;
- создан ли ADR по модели организаций и проектов;
- нет ли противоречий в `ARCHITECTURE.md`.

## Правило закрытия чата

Перед тем как считать чат обработанным, нужно проверить:

- все важные решения перенесены в `docs/project/DECISIONS.md`;
- актуальная архитектура перенесена в `docs/ARCHITECTURE.md`;
- ближайшие задачи перенесены в `docs/project/ROADMAP.md`;
- правила для AI-агентов перенесены в `AGENTS.md`;
- при необходимости создан файл в `docs/modules/`;
- изменения проверены через `git diff`;
- сделан commit.

После этого статус чата можно изменить на:

```text
documented
```

или:

```text
archived
```
