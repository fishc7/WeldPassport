# Архитектурные решения WeldPassport

Журнал ключевых архитектурных решений проекта. Каждая запись фиксирует итог
согласованной ревизии или явного решения владельца. Канон архитектуры —
`docs/ARCHITECTURE.md`; контекст домена — `docs/00_PROJECT_CONTEXT.md`.

---

## 2026-07-01 — Завершение архитектурной ревизии

**Дата:** 2026-07-01

**Контекст:** завершена независимая архитектурная ревизия проекта; выводы
подтверждены тремя согласованными анализами (ChatGPT, Cursor Agent, Claude).

**Принятые решения:**

- Backend принят как основная архитектура.
- Документация больше не должна опережать код.
- Начинается реализация вертикальных бизнес-сценариев.
- Первый вертикальный сценарий — полный жизненный цикл сварного стыка.
- ETL откладывается.
- Import Pipeline остаётся внутри Backend.
- Cursor используется для реализации.
- Архитектурные решения принимаются только после ревизии.

**Связанные материалы:** `docs/ARCHITECTURE.md` (раздел «Архитектурная веха —
2026-07-01»).

## 2026-07-01 — Изменение стратегии разработки

**Дата:** 2026-07-01

**Контекст:** после независимой архитектурной экспертизы (ChatGPT, Cursor Agent,
Claude) пересмотрен порядок вертикальной реализации. Ранее первым сценарием
предполагался полный жизненный цикл сварного стыка (см. запись «Завершение
архитектурной ревизии» выше).

**Принятые решения:**

- Мы изменяем порядок разработки.
- Первым вертикальным сценарием становится полный жизненный цикл работника.
- Вторым вертикальным сценарием станет полный жизненный цикл сварного стыка.
- Контур допуска является фундаментом производственной системы WeldPassport.
- Это решение не отменяет центральную роль сварного стыка в производственном
  контуре.

**Причина:** контур допуска является фундаментом всей производственной системы
WeldPassport.

**Связанные материалы:** `docs/ARCHITECTURE.md` (подраздел «Приоритет первого
вертикального сценария» в разделе «Архитектурная веха — 2026-07-01»).
