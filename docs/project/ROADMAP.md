# WeldPassport — дорожная карта

## Назначение

Этот файл фиксирует план развития WeldPassport.

Он нужен, чтобы не терять направление проекта и понимать, что делать дальше.

## Текущий этап

Этап: архитектурное проектирование и подготовка базы проекта.

Основная цель текущего этапа:

- зафиксировать ключевые архитектурные решения;
- определить базовые сущности;
- подготовить структуру документации;
- подготовить правила работы AI-агентов;
- перейти к проектированию базы данных и реализации модулей.

## Ближайшие задачи

### 1. Зафиксировать архитектурные решения

Статус: в работе.

Нужно фиксировать решения в:

```text
docs/project/DECISIONS.md
docs/ARCHITECTURE.md
AGENTS.md
```

### 2. Уточнить модель организаций и проектов

Статус: принято.

Принята модель:

```text
companies ↔ projects через project_companies
```

### 3. Спроектировать модель персонала

Статус: впереди.

Нужно определить сущности:

```text
persons
employees
welders
welder_stamps
welder_certifications
welder_admissions
```

Нужно учесть:

- человека создаёт ОК;
- сварщиком человек становится после проверки ОГС;
- у сварщика могут быть клейма;
- у сварщика могут быть аттестации;
- допуск к проекту может зависеть от проекта, материала, способа сварки, диаметра, толщины и срока действия.

### 4. Спроектировать модель проекта

Статус: впереди.

Нужно определить структуру:

```text
project
area
line
system
drawing
isometric
weld_joint
```

### 5. Спроектировать модель сварного соединения

Статус: впереди.

Нужно определить:

- номер стыка;
- связь с проектом;
- связь с изометрией;
- диаметр;
- толщина;
- материал;
- тип соединения;
- способ сварки;
- дата сварки;
- сварщик корня;
- сварщик заполнения;
- статус;
- связь с НК;
- связь с исполнительной документацией.

### 6. Спроектировать модель контроля качества и НК

Статус: впереди.

Нужно определить:

- виды контроля;
- процент контроля;
- результаты контроля;
- лабораторию НК;
- дефекты;
- ремонт;
- повторный контроль;
- связь с периодикой КСС.

### 7. Спроектировать модуль периодики КСС

Статус: впереди.

Нужно учесть правила ГОСТ 32569:

- партия однотипных стыков;
- срок партии не более 3 месяцев;
- лимит стыков по группам DN;
- один сварщик;
- одна марка стали;
- единый процесс сварки;
- толщина в пределах ±50%;
- удвоение контроля при браке.

### 8. Подготовить ER-диаграмму

Статус: впереди.

Нужно построить первую ER-модель:

```text
companies
projects
project_companies
persons
welders
welder_certifications
project_members
isometrics
weld_joints
ndt_reports
documents
```

### 9. Перейти к реализации

Статус: после архитектурного ревью.

Перед кодом нужно:

- проверить `docs/ARCHITECTURE.md`;
- проверить `docs/project/DECISIONS.md`;
- проверить `AGENTS.md`;
- сформировать задачу для Cursor Agent или Claude Code;
- после изменений смотреть `git diff`;
- делать commit только после проверки.

## Правило изменения архитектуры

Нельзя менять ключевую архитектуру без записи в `docs/project/DECISIONS.md`.

Если появляется новое важное решение, нужно добавить новый ADR.
